# Как внести вклад

Спасибо за интерес к проекту. Перед изменением поведения сначала создайте issue
и опишите задачу. Небольшие исправления документации можно отправлять сразу.

## Подготовка

```bash
git clone --recurse-submodules <URL-РЕПОЗИТОРИЯ>
cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Debug
cmake --build build --target tgit-revision-graph
```

## Правила изменений

- сохраняйте приложение read-only по отношению к открываемому репозиторию;
- не заменяйте оригинальную OGDF-раскладку самописной;
- изменения поведения Revision Graph сверяйте с upstream TortoiseGit;
- не добавляйте побочные Git-инструменты без отдельного обсуждения;
- проверяйте сборку и запуск на тестовом репозитории;
- не добавляйте безусловные зависимости от конкретной ОС; платформенный код
  изолируйте и документируйте;
- обновляйте документацию вместе с изменением интерфейса.

## Pull request

Один pull request должен решать одну задачу. В описании укажите причину,
реализацию, способ проверки и приложите снимок экрана для визуальных изменений.
Новый код должен быть совместим с GPLv2 и не приносить несовместимых зависимостей.

Участие в проекте означает согласие с [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
