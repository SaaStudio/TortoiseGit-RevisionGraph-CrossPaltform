# Политика безопасности

## Поддерживаемые версии

Исправления безопасности выпускаются только для последней версии основной ветки.

## Сообщение об уязвимости

Не публикуйте сведения об уязвимости в обычном issue. Используйте приватное
сообщение об уязвимости на вкладке **Security → Report a vulnerability** данного
GitHub-репозитория. Укажите затронутую версию, способ воспроизведения, влияние и
возможное исправление.

Если private vulnerability reporting недоступен, свяжитесь с владельцем
репозитория через его публичный GitHub-профиль и запросите приватный канал связи.
